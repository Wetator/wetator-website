Prerequisites

  • JDK version 6 or higher
  • If you plan to run Wetator tests from ant than you additionally need Apache Ant version 1.7.0 or newer.
  • an XML editor (maybe the Eclipse plugin Wetator Test Editor) to edit the Wetator test files (Wetator supports several formats, but XML is suggested)

Command Line Version

Installation of the command line version

  • Download the latest stable version.
  • Create a new directory and unzip the file into it (preserve subdirectories).
  • Open a command window and go to your Wetator directory.
  • If under Windows start 
    wetator.bat
    under Mac OS execute 
    java -cp lib/ -jar lib/wetator.jar
  • If you see some message like 
    Starting Wetator
Wetator execution failed: The configuration file 'D:\Wetator\wetator.config' does not exist.
    then everything seems to be right and you can start with your first test.

Run a simple test

  • Copy the file wetator.config from directory samples\wetator_testform to the root of your installation directory.
    This file defines Wetator's setup.
  • Start Wetator again.
  • A small popup window appears. Please choose the file samples\wetator_testform\wetator_testform.wett and click on open (You can select more then one file if you like).
  • Wetator executes all commands from the selected file. A small progress report is written to the console.
  • If everything works fine you will end with some output like 
    Tests run: 8,  Failures: 0,  Errors: 0
  • Wetator writes a detailed protocol for every test run. You can find this protocol in the subfolder 'logs'. Open the file 'run_report.xsl.html' with your browser.

For more details about the command line interface see Command-Line Options.

To run your tests as ant task see Ant Task.
